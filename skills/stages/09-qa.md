# Stage 9 — QA

You are the QA tester. Verify the built blog post visually and functionally before publish.

## Load

- Startup context from `skills/seo-pipeline.md`.
- `blog/[slug].html` or `deploy/blog/[slug]/index.html`.
- Build validation notes from Stage 8.
- `qa-reports/[slug]-images.md` if it exists.

## Process

1. Open the generated page in a browser.
2. Desktop check:
   - Hero/title/meta render correctly.
   - Headings, paragraphs, lists, callouts, tables, CTA, author box, and footer render correctly.
   - Images load and are not distorted.
3. Mobile check around 375px width:
   - No horizontal overflow.
   - Images and tables handle narrow screens.
   - Navigation and CTA remain usable.
4. Link check:
   - Internal links do not 404.
   - Table of contents anchors scroll to the right sections.
   - CTA link is present and correct.
   - External links look valid.
5. Console check:
   - JavaScript errors.
   - Missing images/CSS/JS.
   - Mixed content warnings.
6. Content QA:
   - U+2014 em dash count is 0.
   - Banned terms are absent or only appear in quoted/source contexts.
   - CTA matches the article topic.
   - In-article media opportunity exists beyond the OG card.
7. Performance sanity check:
   - Page loads quickly.
   - Images are not obviously oversized.
